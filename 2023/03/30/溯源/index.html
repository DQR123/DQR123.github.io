IP溯源

溯源过程中切勿用客户的网络去访问，防止对方的VPS上有JS钩子，导致对方拥有我的网络信息，尽量用虚拟机、浏览器无痕等模式进行访问

判断IP状态

  • 静动态判断
    • 国内
      • 存在rDNS为静态 www.ipip.net
      • 查询相关备案信息 ipwhois.cnnic.net.cn
        • ALLOCATED PORTABLE 动态IP，5G/4G/家庭宽带等
        • ALLOCATED NON-PORTABLE 静态IP，专线宽带/IDC机房/企业出口
    • 国外
      • https://www.virustotal.com/gui/search

判断IP是否为CDN

  • 原理
    • 根据 Host 字段判断是否为绑定域名，进行跳转
  • 特征
    • CDN会绑定许多域名
  • 判断
    • 对反绑的域名进行查询 nslookup
      • nslookup -q=cname baidu.com
    • https://tools.ipip.net/cdn.php 查询
    • 访问IP
      • 通过ModHeader 修改 Host 字段值为域名

其他

  • 子域名：子域名的实时解析IP和历史解析IP都可以作为新的IP进行溯源关联

    • teemo — 扫描邮箱
      • python teemo.py -d
    • subDomainsBrute — 扫描真实IP
      • python subDomainsBrute.py –full

IP端口

  • 使用 nmap 扫描端口 ——切勿用客户的网络进行扫描

    • nmap T4 -Pn 192.168.131.1 快速扫描top1000端口
      nmap -Pn -p- -A -sV -oN 123.txt 127.0.0.1 全扫描

    • 根据端口判断是否为跳板机

      • 5555:SoftEther VPN
1723:PPTP VPN
500:L2TP VPN
4500:L2TP VPN
1701:L2TP VPN
1194:Openvpn
5000:Vtun 
12975:	LogMeIn Hamachi
1080:其他SOCKS5类代理常用端口

    • C&C监听端口或黑客工具端口
      50050 Cobalt Strike
      8080 nps 一些自定义木马也喜欢用这种类型的端口
      8834 nessusWeb 界面管理端口
      4444 hfs

  • 对C段资产进行分析

    • nmap -sP -oN 123.txt 192.168.131.0/24
    • 存在大量的 80/443 端口该C段为IDC或企业的概率比较大

案例

161.35.76.1

  • 直接搜索IP，判断国内还是国外
  • 国外：virustotal.com/gui/search
    • 查看VPS厂商 Autonomous System Label
      • Autonomous System Label DIGITALOCEAN-ASN（VPS厂商）
    • 查看关联的信息 ：Relation
      • 有关联的 CS木马信息、邮件钓鱼
    • 报毒情况
      • Detection 52/72 72款杀毒软件、52款报毒
  • 暴露库查询：https://censys.io/ipv4
    • 22 Banner：SSH-2.0-OpenSSH_7.6p1 Ubuntu-4ubuntu0.3
      • 发现该VPS为 Ubuntu
    • 443 HTTPS 中的 SSL 证书：C=US, ST=, L=, O=jQuery, OU=Certificate Authority, CN=jquery.com
      • jquery.com为SSL证书绑定的域名，jquery.com为默认域名

172.105.87.139

  • 国外：virustotal.com/gui/search
    - 查看关联的信息 ：Relation
    - 域名绑定：Domain
